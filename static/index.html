API detail

Get todos

This is a GET request and it is used to “get” data from an endpoint. There is no request body for a GET request.

A successful GET response will have a 200 OK status, and JSON data.

  • Curl
  • HTTP 
curl -X GET "https://todo-7ni6.onrender.com/todos"
GET /todos HTTP/1.1
Host: todo-7ni6.onrender.com

Create Todo

This is a POST request, submitting data to an API via the request body. This request submits JSON data, and the data is reflected in the response.

A successful POST request typically returns a 200 OK or 201 Created response code.

  • Curl
  • HTTP 
curl -X POST -d '{
	"title": "Add TODO"
}' "https://todo-7ni6.onrender.com/todos"
POST /todos HTTP/1.1
Host: todo-7ni6.onrender.com

{
	"title": "Add TODO"
}

This is a GET request and it is used to “get” data from an endpoint. There is no request body for a GET request.

A successful GET response will have a 200 OK status, and JSON data.

  • Curl
  • HTTP

GET /todos/1 HTTP/1.1

Modify Todo

This is a PUT request and it is used to overwrite an existing piece of data. For instance, after you create an entity with a POST request, you may want to modify that later. You can do that using a PUT request. You typically identify the entity being updated by including an identifier in the URL (eg. id=1).

A successful PUT request typically returns a 200 OK.

  • Curl
  • HTTP 
curl -X PUT -d '{
    "title":"wjsbhayuw"
}' "https://todo-7ni6.onrender.com/todos/1"
PUT /todos/1 HTTP/1.1
Host: todo-7ni6.onrender.com

{
    "title":"wjsbhayuw"
}

Delete Todo

This is a DELETE request, and it is used to delete data that was previously created via a POST request. You typically identify the entity being updated by including an identifier in the URL (eg. id=1).

A successful DELETE request typically returns a 200 OK.

  • Curl
  • HTTP 
curl -X DELETE "https://todo-7ni6.onrender.com/todos/1"
DELETE /todos/1 HTTP/1.1
Host: todo-7ni6.onrender.com
